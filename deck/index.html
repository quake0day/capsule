the problem

AI agents reason at the file level. Engineers reason at the subsystem level.

Every coding agent in production today re-derives architecture from raw source on every task. Context windows overflow, contracts get re-invented, and integration work silently breaks.

The bottleneck isn't model capability. It's the absence of a unit AI can read, compose and verify.

capsule

A capsule is a bounded software subsystem that is self‑describing, self‑verifying and addressable — small enough to fit in an agent's head, large enough to ship.

— 01 / bounded
Owns one job.
A capsule maps to one engineering concern — auth, billing, search ranking — not a file, not a repo.
— 02 / composable
Declares its edges.
Every input and output is a typed contract. Capsules wire together through interfaces, not imports.
— 03 / self-verifying
Ships its own tests.
Contract tests, integration tests and agent-readable invariants travel with the capsule.
— 04 / addressable
Has a URL.
capsule://org/name@version — referenceable from any agent, CLI, MCP server, or chat thread.
04 · definition
04 / 13
capsule.yaml — auth-ldap@0.2.1
capsule: wolfctf/auth-ldap
version: "0.2.1"
purpose: "Identity for lab platform"

interfaces:
provides: [user.identity, session.token]
requires: [ldap.endpoint, kv.store]

invariants:
- one user → one stable id
- sessions expire ≤ 12h

verify:
- contract: tests/contract/*
- integration: tests/int/*

agent_brief: docs/agent.md  # 1.8kb
extension_points: [oauth, saml]
anatomy

A capsule is one folder. Six things inside.

  • 01Purpose. Why it exists, in one line.
  • 02Interfaces. What it provides and requires — typed.
  • 03Invariants. What must stay true.
  • 04Code. The actual implementation.
  • 05Verification. Tests the capsule runs against itself.
  • 06Agent brief. A compact, AI-readable handoff doc.

how it works

Three commands. One protocol. Any agent.

— STEP 01
Author a capsule from any repo.
Point the CLI at a directory. It infers interfaces, drafts the brief, generates contract tests, and publishes to the registry.
$ capsule init ./services/auth › detected: auth-ldap › inferred 2 interfaces › wrote capsule.yaml $ capsule publish → capsule://acme/auth-ldap@0.1
— STEP 02
Compose capsules into an agent's context.
Bundle the capsules a task touches. Output is a compact, typed context pack — drops into Claude Code, Codex, Cursor, or any MCP host.
$ capsule bundle \   auth-ldap billing-stripe \   lab-runtime --for claude › 6.2kb context · 0 conflicts → ready: ./bundle.capsule
— STEP 03
Verify before merge — automatically.
Agents change code; capsules check themselves. Contract drift, broken invariants and missing handoff state surface before the PR opens.
$ capsule verify --bundle › auth-ldap    ✓ 42/42 › billing-stripe ✓ 28/28 › contract drift 0
